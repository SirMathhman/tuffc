# Spec 11: If-Statements and If-Expressions

## Summary

Support conditional control flow via `if (condition) ... else ...` in both statement and expression forms.

## Grammar

```
statement      := ... | if_statement
if_statement   := 'if' '(' expression ')' statement ['else' (statement | if_statement)]

expression     := ... | if_expression
if_expression  := 'if' '(' expression ')' expression 'else' (expression | if_expression)
```

## Semantics

### If-Statement

**Syntax**: `if (condition) then_stmt [else else_stmt]`

- `condition` must have type `Bool`
- `then_stmt` is a statement: assignment, let, statement-block, or nested if-statement
- `else` clause is **optional**
- If `else` is present, `else_stmt` is a statement or another if-statement (for else-if chaining)
- No value is produced (used for side effects only)
- Valid at program end (program exits 0)

**Examples**:
```
if (x < 5U8) y = 10U8;

if (cond) { let z = 1U8; } else { x = 2U8; }

if (a) {
  x = 1U8;
} else if (b) {
  x = 2U8;
} else {
  x = 3U8;
}

if (true) { }  // empty statement block
```

### If-Expression

**Syntax**: `if (condition) then_expr else else_expr`

- `condition` must have type `Bool`
- `then_expr` and `else_expr` are expressions: literals, variables, arithmetic, block-exprs, nested if-exprs
- `else` clause is **REQUIRED**
- Produces a value with type determined by type-checking rules below

**Type-checking**:
1. If assigned to a declared type `T`, both branches must be type-compatible with `T`
2. If no declared type, infer the **common type** that covers both branch types
   - For integer types: use the wider type (e.g., `U8` and `U16` → `U16`)
   - For incompatible types (e.g., `U8` and `Bool`): type error
3. Both branches must have the same inferred/declared type after compatibility checking

**Examples**:
```
if (true) 5U8 else 10U8  // type: U8, evaluates to 5U8

let x: U16 = if (c) 5U8 else 10U16;  // both U8 and U16 compatible with U16

let y = if (c) 5U8 else 10U16;  // infer common type U16

if (c) { let z = 1U8; z + 1U8 } else 10U8  // block-expr as then-branch

if (a) 1U8 else if (b) 2U8 else 3U8  // else-if chaining in expression
```

## Else-If Chaining

Both if-statements and if-expressions support `else if` as syntactic sugar:

```
if (c1) ... else if (c2) ... else if (c3) ... else ...
```

This is parsed as nested if-statements/expressions without requiring braces around the nested if.

## Invalid Forms

```
if (c) 5U8  // ERROR: expression branch with no else

if (c) { 5U8 }  // ERROR: block-expr as then-branch with no else
                // (can't be statement because ends with expr,
                //  can't be expression because no else)

if (5U8) x = 1U8;  // ERROR: condition must be Bool

let x = if (c) 5U8 else true;  // ERROR: no common type (U8 vs Bool)

let x: U8 = if (c) 5U8 else 256U16;  // ERROR: U16 not compatible with U8
```

## User Stories

### If-Statement
1. As a programmer, I want to conditionally execute code based on a Bool, so that I can implement side-effectful conditional logic
2. As a programmer, I want to execute one of two statement branches, so that I can handle different cases
3. As a programmer, I want to chain else-if conditions, so that I can handle multiple cases
4. As a programmer, I want to use empty statement blocks in if-branches, so that I can stub out logic

### If-Expression
1. As a programmer, I want to compute a value based on a Bool condition, so that I can initialize variables conditionally
2. As a programmer, I want the compiler to infer a common type for compatible branches, so that I can mix U8/U16/etc naturally
3. As a programmer, I want to use block expressions in if-branches, so that I can perform statements before computing the branch value
4. As a programmer, I want to chain else-if in expressions, so that I can select from multiple values
5. As a programmer, I want to use if-expressions in arithmetic/comparisons, so that I can compose conditional logic

## Codegen

**If-statement**:
```
if (condition_code) {
  then_stmt_code
} else {
  else_stmt_code  // if else present
}
```

**If-expression**:
```
(condition_code ? then_expr_code : else_expr_code)
```

For block-expr branches, the IIFE is generated by the block-expr parser, so:
```
(condition_code ? (() => { stmts; return value; })() : else_code)
```
